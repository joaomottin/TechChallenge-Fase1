# Tech Challenge - Fase 1

## Sobre o projeto

Este repositório contém o trabalho acadêmico desenvolvido para o **Tech Challenge - Fase 1** da FIAP, com base no case de e-commerce da Olist.

O objetivo do projeto é transformar dados transacionais em uma análise executiva voltada para tomada de decisão. A partir da base pública da Olist, o grupo avaliou o desempenho de vendedores, com foco na relação entre satisfação dos clientes, volume de pedidos, receita e risco para a reputação da plataforma.

## Contexto da análise

A Olist atua como um ecossistema que conecta vendedores a grandes marketplaces. Nesse modelo, a experiência entregue por cada parceiro pode impactar a percepção dos clientes sobre a marca como um todo.

Por isso, a análise busca responder uma pergunta central: vendedores com baixa satisfação representam apenas casos isolados ou indicam um risco relevante para a confiança no ecossistema?

## Escopo

O estudo utiliza dados de pedidos, itens, avaliações e vendedores para:

- consolidar uma visão por vendedor;
- classificar vendedores detratores com base na média de avaliações;
- comparar detratores e não detratores em pedidos, receita e participação na base;
- identificar faixas de atenção para apoiar recomendações de gestão;
- construir gráficos e indicadores para o relatório executivo.

## Arquivos principais

- `Analise_vendedores_baixa_nota_V01.ipynb`: notebook com a preparação dos dados, limpeza, consolidação das métricas, segmentação dos vendedores e geração dos gráficos.
- `data/`: pasta com os arquivos CSV utilizados na análise.

## Base de dados

A análise foi desenvolvida com o **Brazilian E-Commerce Public Dataset by Olist**, base pública com informações de pedidos, vendedores, produtos, pagamentos, avaliações e entregas do e-commerce brasileiro.
